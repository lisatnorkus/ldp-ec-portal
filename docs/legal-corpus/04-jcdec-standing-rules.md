# JCDEC Standing Rules and Resolutions

JCDEC-level rules adopted by the body under `[LJCDP §25]` (new business) or by majority vote at any meeting with quorum. Standing rules persist until rescinded by a subsequent CEC vote.

## What goes here vs elsewhere

| Type of rule | Where it lives | Authority |
|---|---|---|
| Rules in the LJCDP bylaws themselves | `03-ljcdp-bylaws.md` and the verbatim source | Adopted/amended under `[LJCDP §29.1]` |
| LDP-wide policies (endorsement process, code of conduct, fiscal policy) | Adjacent corpora or this file | Adopted by the CEC or relevant Committee |
| Committee operating rules (Finance Fiscal Policy, Endorsement Process Committee rules) | This file or in committee minutes | Adopted by that committee + ratified by CEC where required |
| Robert's Rules of Order | Applies as fallback per `[LJCDP §28.1]` | Robert's Rules of Order Newly Revised, 12th ed. |
| KDP-level rules | `02-kdp-bylaws.md` | KDP Bylaws |
| DNC-level rules | `01-dnc-charter.md` | DNC Charter + Bylaws |

## How to add a standing rule to this file

When a CEC vote adopts or amends a standing rule:

1. Capture the **exact motion language** from the meeting minutes.
2. Note the **vote and date** ("Adopted by JCDEC 2026-09-15, 42–3–1").
3. Note **who proposed it** and **what authority anchors it** (bylaw section, committee chair authority, etc.).
4. Add as a numbered section in this file.
5. Update `MEMORY.md` index if the rule changes the answer to a recurring question.

Citation format:
```
[JCDEC Standing Rule, adopted 2026-09-15]
[JCDEC Resolution 2026-04, adopted 2026-04-23]
[LDP Endorsement Process Policy, adopted YYYY-MM-DD]
[LJCDP Fiscal Policy 2026, adopted 2026-01-15]
```

## Maintenance

This file is updated by the JCDEC Secretary (per `[LJCDP §13]`) or designee whenever a standing rule is adopted, amended, or rescinded. If the rule is bylaw-mandated, also reflect the change in the bylaws-side files (`03-ljcdp-bylaws.md`).

For the LDP Endorsement Process specifically, see `06-endorsement-process.md` — that file has the operational treatment; this file would carry the verbatim adopted policy text once it's committed.
